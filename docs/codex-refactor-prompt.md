请接手这个 COC 车卡工具项目，先不要新增功能页，优先做“拆分后的二次重构 + 性能优化”。

项目目标：面向中文 COC 玩家、手机端优先的轻量车卡工具。当前阶段只做本地网页原型，不接服务器、不接 AI、不做账号系统、不做付费/商业化功能。

请先阅读 `AGENTS.md`、`README.md` 和现有文件结构。当前已经从单文件 HTML 拆成：

- `index.html`
- `css/styles.css`
- `data/occupations.js`
- `data/skills.js`
- `js/app.js`

现在请继续把 `js/app.js` 拆成更可维护的模块，并做性能优化。

目标文件结构建议：

```text
js/app.js                  入口，只负责初始化
js/state.js                全局状态、常量、工具函数
js/storage.js              localStorage 保存/恢复，加入 300ms 防抖
js/router.js               步骤切换、顶部步骤显示
js/profile.js              基础信息页
js/attributes.js           属性购点页、随机属性、次要属性计算
js/skills.js               技能加点页、职业联动、技能排序筛选、信用评级计入职业点
js/background-items.js     背景&物品页
js/preview.js              档案预览浮窗，折叠区按需渲染
js/notes.js                备忘笔记浮窗
js/modal.js                弹窗通用逻辑
```

数据文件保持独立：

```text
data/occupations.js
data/skills.js
```

重要要求：

1. 不要改变用户已经确认过的页面文案和步骤名。步骤固定为：基础信息、属性购点、技能加点、背景&物品、头像&立绘、角色预览。
2. 不要新增服务器、构建工具、框架、账号系统、AI 功能。保持原生 HTML/CSS/JS。
3. 重构后页面必须仍然可以直接打开 `index.html` 使用。
4. 保留 localStorage 数据结构兼容性；确实需要迁移时，请写兼容读取逻辑。
5. 技能页不要显示困难成功/极难成功。
6. 信用评级只保留顶部输入框，并计入职业点数；不要在技能列表里重复显示“信用评级”技能。
7. 技能默认排序保持：本职技能优先、常用类标签优先、母语、克苏鲁神话、其他技能；其他技能按行动、知识、技能显示顺序排序。
8. 档案预览浮窗默认只展开基础信息；其他折叠区按需渲染，尤其技能预览不要在每次输入时全量重算。
9. 技能列表渲染请优化：使用事件委托，避免每个输入框重复绑定大量监听；筛选/排序/数据变化时再重绘。
10. localStorage 保存请加防抖，避免每输入一个字符就写入完整数据。
11. 减少可能导致卡顿的重绘；不要在输入事件中反复重建整个页面。
12. 保持 UI 风格：灰白底、藏蓝字、清爽档案表、手机端优先。

- 如果有无法确认的地方，请明确标注。
