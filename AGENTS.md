# AGENTS.md

## 项目定位

这是一个面向中文 COC 玩家、手机端优先的轻量车卡工具。第一阶段是本地 HTML/CSS/JS 原型，不接服务器、不接 AI、不做账号系统、不做付费系统。

目标是替代 Excel 车卡流程，让玩家能更舒服地在手机或电脑上创建、查看、编辑调查员角色卡。

## 固定步骤

页面步骤固定为：

1. 基础信息
2. 属性购点
3. 技能加点
4. 背景&物品
5. 头像&立绘
6. 角色预览

顶部主标题统一为「COC车卡工具」。右侧显示当前步骤，例如「3 / 6　技能加点」。

## UI 风格

- 灰白底、藏蓝文字、清爽档案表风格。
- 手机端优先，但桌面端也要好用。
- 不显示开发说明，例如“第一版”“后续会做”等。
- 不要把页面做成 TRPG 大平台风格，也不要过早出现商业化、账号、价格、导出等内容。
- 输入框尽量不要写多余 placeholder，除非是浮窗笔记或复杂字段需要提示。

## 功能边界

- 使用 localStorage 保存。
- 不接 AI。
- 不接服务器。
- 不做账号系统。
- 不做复杂导出。
- 规则优先按上传的 COC7 空白卡 Excel 与后续提供的规则资料处理。

## 技能页约定

- 技能页不显示困难成功/极难成功。
- 技能显示：技能名、初始值、职业加点、兴趣加点、合计值。
- 信用评级使用顶部单独输入框，计入职业点数，不在技能列表中重复显示。
- 本职、探索、交涉、战斗、心理、行动、知识、技能等都作为标签，不做互斥分组。
- 默认排序：本职技能优先、常用类标签优先、母语、克苏鲁神话、其他技能；其他技能按行动、知识、技能排序。

## 档案预览约定

- 档案预览是浮窗。
- 默认只展开基础信息。
- 大项可折叠。
- 需要使用两列/紧凑排布，避免一列过长。
- 完成属性购点后显示 HP、SAN、MP、DB、Build、MOV。
- 完成技能加点后显示合计值超过 30 的技能，排序规则和技能页一致。

## 待优化记录

- 属性名和技能名支持点击/停留显示注释文本。
- 添加计算“特长数”的功能。
- 技能框显示设计和大小后续优化，但不优先。
- 档案预览继续优化为更紧凑排布。

## 重构要求

进行重构时要保持现有功能可用，避免一次性大改。优先拆分模块、减少重复渲染、减少频繁 localStorage 写入。

建议方向：

- `js/storage.js`：localStorage、防抖保存、恢复数据。
- `js/router.js`：步骤切换与顶部进度。
- `js/profile.js`：基础信息。
- `js/attributes.js`：属性、随机属性、次要属性。
- `js/skills.js`：技能列表、职业联动、加点统计、排序筛选。
- `js/background-items.js`：背景&物品。
- `js/preview.js`：档案预览浮窗。
- `js/notes.js`：备忘笔记浮窗。
- `data/occupations.js`、`data/skills.js` 继续作为纯数据层。

每次修改后请至少手动检查：

- 页面能正常打开。
- 6 个步骤能切换。
- localStorage 保存/恢复正常。
- 职业选择能带到技能页。
- 技能加点统计正常。
- 档案预览能打开且不卡顿。
